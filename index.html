Projects
Formfor.me

Formfor.me is PennApps XII's Best Social/Local Impact & Government Hack. We take IRS forms in PDF format, parse the form into plain text and apply foreign-language translation. We then display the form as easy-to-read questions to the user that they answer in simple one-line responses. Finally, we auto-populate the PDF form and provide the completed form to the user.

Python & Flask, Microsoft Azure, Angular.js, Express.js, Node.js
Travelbot

TravelBot is a web tool that creates a customized itinerary for the city of Chicago based on user feedback. TravelBot presents the user with a possible tourist destination and the user decides whether it is a place they would like to visit. Based on these choices, TravelBot becomes more attuned to the user's preferences and suggests places that it thinks the user would enjoy.

Made using Python, MongoDB, Node.js, Angular.js, Bootstrap, Sass
Top 100 Predictor

Top 100 Predictor is a project that analyzes which ML classifiers best predict the projected popularity of a song. For our project, we took Spotify's Top 100 songs from 2014 and pulled 1,000+ other songs from 2014 so we could see the accuracy of our results. We utilized the Spotify & Echonest song APIs for our song data and Weka, a collection of ML classifiers. Naive Bayes outperformed K* and Decision Trees with the greatest amount of true hits (songs correctly classified as hits), although Decision Trees classified the greatest amount of true misses (songs correctly classified as non-hits).

Made using Weka, Python, Pure.css
The Gibbot Project

I was the lead GUI developer for the Gibbot Project, a project by the Neuroscience and Robotics (NxR) Lab at Northwestern University. My main task was to build a GUI with Java that teaches kids about the Gibbot robot, which models the arm movements of a Gibbon monkey. Additionally, I was in charge of holding weekly GUI team meetings to ensure each member had deliverables and were able to complete them on time. Lastly, I conducted trials with children to test the GUI and its usability.

Made using Java and its graphic libraries (java.awt)
BuzzKill

Built at HackIllinois 2015, BuzzKill is a Pebble smartwatch application that detects what website you are visiting on your laptop, sees whether it is a time-wasting website (e.g. Facebook, Twitter, Imgur, etc.), and if it is, starts a timer on the Pebble that tracks how long you stay on the webpage. Additionally, the Pebble buzzes every 15 seconds to remind you that you are still wasting time on that page. When you exit the time-wasting website and go to a kosher website (e.g. Google, Wikipedia), the timer and incessant buzzing stop.

Made using CloudPebble (C), Firebase, and the Pebble Classic
HeartFlash

Built at TreeHacks 2015, HeartFlash is an iOS app that uses the iPhone flashlight to monitor one's heartbeat and suggest exercises based on the bpm and user-entered age. The user places their fingure over the flash for 15-20 seconds. Once the bpm is calculated, the user is presented with exercises that will raise their bpm to a certain number as well as with explanations on how to perform said exercises.

Made using Swift, Objective-C, and Xcode's storyboard
Shake N' Quake

Shake N' Quake is an HTML5 game that won 1st place at the International Women's Hackathon at Northwestern University in 2014, hosted by Microsoft. The game's purpose is to educate disaster-relief organizations and communities on what is the appropriate protocol during an earthquake. Our game has three levels - Phase 1: Collect items for your inventory before the earthquake, Phase 2: Avoid falling objects and find shelter during the earthquake, and Phase 3: Avoid hazards and find the nearest Red Cross Disaster-Relief tent after the earthquake.

Made using Construct 2 and HTML5
